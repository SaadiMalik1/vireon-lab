# API & Interfaces

NeuroShield offers a robust suite of interfaces for controlling the virtual laboratory, orchestrating simulations, and consuming generated physiological data.

## 1. Command Line Interface (CLI)

The primary entry point for launching headless or UI-backed simulations is the Python CLI.

```bash
python3 -m neuroshield run [OPTIONS]
```

### Key Arguments
- `--duration <seconds>`: Length of the simulation run.
- `--secure-mode`: Activates the `NeuroIDS` and the associated threat intelligence processing. Without this flag, the platform acts only as a signal simulator.
- `--no-report`: Bypasses the generation of the PDF executive summary at the end of the simulation.
- `--lsl`: Bypasses the internal Web UI WebSocket server and instead broadcasts raw multiplexed EEG data directly to the Lab Streaming Layer (LSL). Useful for integrating with third-party software like OpenViBE.
- `--board {synthetic,pieeg,cyton,muse}`: Defines the hardware profile of the Digital Twin (which determines channel count, ADC limits, and noise baselines).

## 2. Model Context Protocol (MCP) Server

NeuroShield can act as an MCP server, allowing Large Language Models and external orchestration scripts to interactively query and drive the digital twin.

The server (`mcp_server.py`) exposes three primary tools:

### `inspect_twin_state`
Returns the complete frozen snapshot of the current `DigitalTwin` state.
- **Output:** JSON object containing `battery_level`, `clinical_status`, `hazard_state`, `electrode_impedances`, etc.

### `mock_authenticate_session`
Generates a valid capability token required to run simulations.
- **Parameters:** `biomarker_hash` (string), `role` (string: "patient" or "clinician").
- **Warning:** As explicitly noted in the documentation and codebase, this is a **mock** function. It simulates a Post-Quantum Cryptographic key exchange for the purpose of the digital twin scenario; it does not perform actual cryptographic operations.

### `run_simulation`
Triggers the `Coordinator` to spin up a simulation run with specific attacks.
- **Parameters:** `session_token` (string), `duration_sec` (float), `attacks` (list of strings).
- **Requires:** A valid session token generated by `mock_authenticate_session`.

## 3. Telemetry Interfaces

NeuroShield broadcasts its state asynchronously while the ReplayEngine is ticking.

### WebSocket API
When launched without the `--lsl` flag, the `Coordinator` spawns an asynchronous WebSocket server on `localhost:8050`.

The Web UI connects to this port and receives three types of JSON packets:
- **`state`**: Physical metrics (battery, temp, impedance) and clinical status.
- **`eeg`**: High-frequency multiplexed signal data arrays.
- **`threat`**: Any `qTARA` enriched anomalies detected by the `NeuroIDS`.

### Lab Streaming Layer (LSL)
When the `--lsl` flag is provided, the WebSocket server is disabled. Instead, NeuroShield creates a standard `StreamInfo` endpoint (`name='NeuroShield_Out', type='EEG'`). 

This allows clinical research tools to subscribe to the simulated output just as they would a physical OpenBCI or Muse device, facilitating end-to-end pipeline testing without physical hardware.
