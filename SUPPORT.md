# Support

**Audience**: End users, Developers, Academic Researchers

## Where to get help

If you are having trouble with VIREON, there are several ways to get support:

1. **Read the Documentation**: The [VIREON Documentation](docs/index.md) contains architecture overviews, API guides, and glossaries that answer most common questions.
2. **GitHub Discussions**: Use the GitHub Discussions tab to ask questions, share ideas, and connect with other researchers. This is the preferred method for general usage questions.
3. **Issue Tracker**: If you believe you have found a bug in the code, or have a specific feature request, please open an issue on the [GitHub Issue Tracker](https://github.com/SaadiMalik1/Vireon/issues). Please refer to our [Contributing Guidelines](CONTRIBUTING.md) before submitting.

## Security Vulnerabilities
If you have found a security vulnerability in the VIREON framework, **do not open a public issue**. Please refer to our [Security Policy](SECURITY.md) for instructions on responsible disclosure.

## Academic & Research Collaboration
If you are an academic researcher looking to collaborate on a paper or integrate VIREON into your lab, please reach out via GitHub Discussions or email the core maintainers directly.
